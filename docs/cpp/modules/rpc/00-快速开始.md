# 00-快速开始

## 依赖

- Galay 内部依赖由顶层 CMake 统一提供：
  - `galay-kernel`
  - `galay-utils`
  - `galay-http`（推荐）
- 支持 C++23 的编译器（推荐 GCC 11+、Clang 14+、AppleClang 15+）
- CMake 3.20+

## 拉取与编译

```bash
git clone https://github.com/gzj-creator/galay.git
cd galay
cmake -S . -B build \
  -DCMAKE_BUILD_TYPE=Release \
  -DGALAY_BUILD_RPC=ON
cmake --build build --target galay-rpc --parallel
```

当前仓库统一由顶层 CMake 提供内部依赖 target。安装后只导出 `galay` 一个 CMake package；外部项目通过 `find_package(galay CONFIG REQUIRED)` 后链接 `galay::rpc`。

如果已安装的 galay 不在默认安装前缀，可显式指定：

```bash
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release \
  -DCMAKE_PREFIX_PATH="/path/to/galay/install"
```

## CMake 开关

- `BUILD_TESTING`：是否构建测试（默认 ON，标准 CTest 开关）
- `GALAY_BUILD_BENCHMARKS`：是否构建基准测试（默认 ON）
- `GALAY_BUILD_EXAMPLES`：是否构建示例（默认 ON）
- `GALAY_ENABLE_CPP23_MODULES`：是否在工具链满足时为 `galay::rpc` 注册 C++23 module file set（默认 OFF）
- `GALAY_RPC_ENABLE_ETCD`：是否把 RPC etcd 服务发现适配编译进 `galay::rpc`（默认 OFF，要求 `GALAY_BUILD_ETCD=ON`）
- `GALAY_DISABLE_IOURING`：Linux 下是否禁用 `io_uring` 并固定使用 `epoll`（默认 ON）

说明：`GALAY_ENABLE_CPP23_MODULES=ON` 只有在 CMake >= 3.28、Ninja 或 Visual Studio 生成器、且编译器支持 CMake C++ module 扫描时，才会在 `galay-rpc` / `galay::rpc` 上注册 module file set。当前 AppleClang 路径会明确跳过 module file set，继续使用 include 路径。
说明：`galay::rpc` 是默认消费 target；Linux 想启用 `io_uring` 时，需要显式传入 `-DGALAY_DISABLE_IOURING=OFF` 并提供 `liburing`。

## 运行示例

### Echo 示例（四种 RPC 模式）

```bash
# 终端 1：启动服务器
./build/examples/e1_echo 9000

# 终端 2：运行客户端
./build/examples/e2_echo 127.0.0.1 9000
```

该链路由 `EchoExampleSmokeTest` 自动验证；命令默认从仓库根目录执行。

### Stream 示例（真实流协议）

```bash
# 终端 1：启动流服务器
./build/examples/e3_stream 9100 1 131072

# 终端 2：运行流客户端
./build/examples/e4_stream 127.0.0.1 9100 200 64
```

该链路由 `StreamExampleSmokeTest` 自动验证；命令默认从仓库根目录执行。

### C++23 模块化导入示例

如果构建了模块示例，可以运行 import 版本：

```bash
# Echo import 版本
./build/examples/e1_echo_import 9000
./build/examples/e2_echo_import 127.0.0.1 9000

# Stream import 版本
./build/examples/e3_stream_import 9100 1 131072
./build/examples/e4_stream_import 127.0.0.1 9100 200 64
```

当 `BUILD_MODULE_EXAMPLES=ON` 且生成器支持模块时，以上命令分别由 `EchoImportExampleSmokeTest` 与 `StreamImportExampleSmokeTest` 自动验证。

## 最小服务端示例

```cpp
#include <galay/cpp/galay-rpc/kernel/rpc_server.h>
#include <galay/cpp/galay-rpc/kernel/rpc_service.h>

using namespace galay::rpc;
using namespace galay::kernel;

class EchoService : public RpcService {
public:
    EchoService() : RpcService("EchoService") {
        registerMethod("echo", &EchoService::echo);
    }

    Task<void> echo(RpcContext& ctx) {
        auto& req = ctx.request();
        ctx.setPayload(req.payload().data(), req.payload().size());
        co_return;
    }
};

int main() {
    RpcServerConfig config;
    config.port = 9000;

    RpcServer server(config);
    server.registerService(std::make_shared<EchoService>());
    server.start();

    while (server.isRunning()) {
        std::this_thread::sleep_for(std::chrono::seconds(1));
    }
}
```

## 最小客户端示例

```cpp
#include <chrono>
#include <galay/cpp/galay-rpc/kernel/rpc_client.h>
#include <galay/cpp/galay-kernel/core/runtime.h>

using namespace galay::rpc;
using namespace galay::kernel;

Task<void> callEcho() {
    RpcClient client;
    if (auto __await_result = co_await client.connect("127.0.0.1", 9000); !__await_result) {
        // 错误处理：记录日志、重试或提前返回
    }

    while (true) {
        auto result = co_await client.call("EchoService", "echo", "Hello");
        if (!result) break;
        if (result.value()) {
            auto& response = result.value().value();
            std::string data(response.payload().begin(), response.payload().end());
            std::cout << "Response: " << data << "\n";
            break;
        }
    }

    co_await client.close();
}

int main() {
    Runtime runtime = RuntimeBuilder().ioSchedulerCount(1).computeSchedulerCount(1).build();
    runtime.start();

    Task<void> task = callEcho();
    auto* scheduler = runtime.getNextIOScheduler();
    scheduler->spawn(task);

    while (!task.done()) {
        std::this_thread::sleep_for(std::chrono::milliseconds(10));
    }
    runtime.stop();
}
```

## 模块化导入版本

使用 C++23 模块时，只需一行导入：

```cpp
import galay.rpc;
```

推荐构建方式（Clang 20 + Ninja）：

```bash
cmake -S . -B build-mod -G Ninja \
  -DGALAY_ENABLE_CPP23_MODULES=ON \
  -DCMAKE_CXX_COMPILER=/opt/homebrew/opt/llvm@20/bin/clang++
cmake --build build-mod --target galay-rpc --parallel
```

## 常见问题

### 编译错误：找不到头文件

如果已安装的 galay 不在默认安装前缀，可通过 `CMAKE_PREFIX_PATH` 显式指定：

```bash
cmake -S . -B build \
  -DCMAKE_PREFIX_PATH="/path/to/galay/install"
```

### 链接错误：undefined reference

检查是否链接了必要的库：

```cmake
target_link_libraries(your_target PRIVATE galay::rpc galay::kernel)
```

### 运行时错误：连接失败

1. 检查服务器是否运行
2. 确认端口号正确
3. 检查防火墙设置
4. 查看服务端日志

### 模块编译失败

当前模块编译路径要求：

- CMake >= 3.28
- Ninja 或 Visual Studio 生成器
- 支持 C++23 模块的编译器

不满足条件时会自动降级到 include 路径。
