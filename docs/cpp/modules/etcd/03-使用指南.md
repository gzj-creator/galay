# 03-使用指南

本页聚焦“怎么正确使用当前 API”，不重复架构细节和完整示例清单。

## 1. 先选同步还是异步

| 维度 | `EtcdClient` | `AsyncEtcdClient` |
| --- | --- | --- |
| 适用场景 | 工具脚本、低并发程序、阻塞式服务 | 协程服务、需要和 `galay-kernel` runtime 集成的场景 |
| 依赖 | 无需 `IOScheduler` | 必须提供 `IOScheduler*` |
| 连接语义 | `connect()` / `close()` 阻塞执行 | `co_await connect()` / `co_await close()` |
| 超时语义 | `request_timeout` 可影响 connect 与请求 | `request_timeout` 用于请求 awaitable，不是公开的 connect 超时 |

如果多个调用方需要复用多端点连接，请使用 `EtcdClusterClient` 或 `AsyncEtcdClusterClient` 租约池。

## 2. 配置规则

### 2.1 endpoint

当前实现接受这些形式：

- `http://127.0.0.1:2379`
- `http://127.0.0.1`
- `http://[::1]:2379`

当前实现拒绝或不建议这些形式：

- `127.0.0.1:2379`：缺少协议
- `https://127.0.0.1:2379`：语法能解析，但当前客户端不支持 TLS 传输
- 依赖 endpoint 路径携带 API 前缀：请改用 `api_prefix`

### 2.2 `api_prefix`

默认值是 `/v3`。源码会自动规范成：

- 有前导 `/`
- 无尾部 `/`

例如：

- `"v3"` -> `"/v3"`
- `"/v3/"` -> `"/v3"`

### 2.3 `request_timeout`

推荐显式设置，以避免把默认“无限等待”带入生产环境：

```cpp
galay::etcd::EtcdConfig sync_config = galay::etcd::EtcdConfig::withTimeout(
    std::chrono::seconds(5));

galay::etcd::EtcdConfig async_config = galay::etcd::EtcdConfig::withTimeout(
    std::chrono::seconds(5));
```

## 3. 同步客户端基本模式

```cpp
galay::etcd::EtcdConfig config;
config.endpoint = "http://127.0.0.1:2379";

auto client = galay::etcd::EtcdClientBuilder().config(config).build();
if (auto conn = client.connect(); !conn) {
    std::cerr << conn.error().message() << '\n';
    return;
}

if (auto put = client.put("/app/key", "value"); !put) {
    std::cerr << put.error().message() << '\n';
    return;
}

if (auto get = client.get("/app/key"); !get) {
    std::cerr << get.error().message() << '\n';
    return;
}

for (const auto& kv : get.value()) {
    std::cout << kv.key << " => " << kv.value << '\n';
}

(void)client.close();
```

使用要点：

- 任何操作前都先 `connect()`
- 先检查返回值，再直接使用 `value()`
- 结束时显式 `close()`

## 4. 同步 cluster client 基本模式

```cpp
galay::etcd::EtcdProductionConfig production;
production.endpoints = {
    "http://127.0.0.1:2379",
    "http://127.0.0.1:22379",
    "http://127.0.0.1:32379",
};
production.connections_per_endpoint = 2;

auto pool = galay::etcd::EtcdClusterClientBuilder()
    .productionConfig(production)
    .requestTimeout(std::chrono::seconds(2))
    .build();

auto put = pool.withClient([](galay::etcd::EtcdClient& client) {
    return client.put("/app/key", "value");
});
if (!put) {
    std::cerr << put.error().message() << '\n';
    return;
}
```

使用要点：

- `production.endpoints` 为空时会回落到 `EtcdConfig::endpoint`
- `connections_per_endpoint` / `connectionsPerEndpoint()` 控制每个 endpoint 的 client 数量
- `withClient()` 会自动获取租约并确保 client 已连接；操作成功或失败后都会自动归还
- 池空时返回 `PoolExhausted`，建连与操作错误通过同一个 `std::expected<T, EtcdError>` 返回

需要让同一 client 连续执行多个操作时，可使用进阶手动路径：

```cpp
auto lease = pool.acquireConnected();
if (!lease) {
    std::cerr << lease.error().message() << '\n';
    return;
}

auto put = lease->get()->put("/app/key", "value");
if (!put) {
    std::cerr << put.error().message() << '\n';
    return;
}
auto get = lease->get()->get("/app/key");
// lease 离开作用域时自动归还；也可调用 lease->release() 提前归还。
```

如果必须把“获取”和“连接”完全拆开，仍可使用低阶 `tryAcquire()`；它不等待、不连接网络。

## 5. 异步客户端基本模式

```cpp
Task<void> run(galay::kernel::IOScheduler* scheduler) {
    galay::etcd::EtcdConfig config;
    config.endpoint = "http://127.0.0.1:2379";

    auto client = galay::etcd::AsyncEtcdClientBuilder()
        .scheduler(scheduler)
        .config(config)
        .build();

    if (auto conn = co_await client.connect(); !conn) {
        std::cerr << conn.error().message() << '\n';
        co_return;
    }

    if (auto put = co_await client.put("/app/key", "value"); !put) {
        std::cerr << put.error().message() << '\n';
        co_return;
    }

    if (auto get = co_await client.get("/app/key"); !get) {
        std::cerr << get.error().message() << '\n';
        co_return;
    }

    (void)co_await client.close();
}
```

使用要点：

- `IOScheduler*` 不能为空
- 完整可运行版本请直接用 `examples/etcd/include/e2_basic.cc`
- `co_await` 后直接使用 `value()`，不再依赖最近结果缓存

## 6. Async cluster client 池

```cpp
Task<void> runPool(galay::kernel::IOScheduler* scheduler) {
    galay::etcd::EtcdProductionConfig production;
    production.endpoints = {
        "http://127.0.0.1:2379",
        "http://127.0.0.1:22379",
    };
    production.connections_per_endpoint = 2;

    auto cluster = galay::etcd::AsyncEtcdClusterClientBuilder()
        .scheduler(scheduler)
        .productionConfig(production)
        .apiPrefix("/v3")
        .requestTimeout(std::chrono::seconds(2))
        .build();

    auto task_result = co_await cluster.withClient(
        [](galay::etcd::AsyncEtcdClient& client)
            -> galay::kernel::Task<galay::etcd::EtcdGetResult> {
            co_return co_await client.get("/app/key");
        });
    if (!task_result) {
        std::cerr << task_result.error().message() << '\n';
        co_return;
    }
    auto result = std::move(task_result.value());
    if (!result) {
        std::cerr << result.error().message() << '\n';
        co_return;
    }
}
```

使用要点：

- 所有池内 client 固定绑定 builder 指定的同一个 `IOScheduler`
- `withClient()` 会把操作和租约保存到协程 frame，自动连接并保证任意返回路径归还 client
- `co_await Task<T>` 的最外层是 `std::expected<T, TaskResultError>`；先检查 Task 层，再检查内部 `EtcdError`
- 回调 Task 的调度/消费错误会映射成 `EtcdErrorType::Internal`，原消息会保留
- pool 不执行透明 retry 或后台连接维护

需要在同一租约内连续执行多个异步操作时，使用进阶路径：

```cpp
auto acquire_task = co_await cluster.acquireConnected();
if (!acquire_task) {
    co_return; // TaskResultError
}
auto lease = std::move(acquire_task.value());
if (!lease) {
    co_return; // EtcdError
}

auto put = co_await lease->get()->put("/app/key", "value");
if (!put) {
    co_return;
}
auto get = co_await lease->get()->get("/app/key");
// lease 离开协程作用域时自动归还。
```

`tryAcquire()` 仍是最低阶、立即返回且不做网络 I/O 的入口；调用方必须自行保证租约不跨 scheduler，并自行完成连接。

## 7. 常见操作模式

### 7.1 前缀查询与前缀删除

```cpp
auto get = client.get("/config/service/", true, 100);
auto del = client.del("/config/service/", true);
```

注意：

- `prefix=true` 是按字节前缀匹配，不是“路径感知”语义
- 是否带尾部斜杠由你的业务命名规范决定，不是库强制要求
- 大前缀查询建议总是带 `limit`

### 7.2 租约

```cpp
if (auto lease = client.grantLease(30); !lease) {
    // 失败处理
}

int64_t lease_id = lease.value();
if (auto put = client.put("/service/node-1", "online", lease_id); !put) {
    // 失败处理
}

if (auto keepalive = client.keepAliveOnce(lease_id); !keepalive) {
    // 失败处理
}
```

注意：

- `grantLease()` 只返回租约 ID；绑定键时要手动把 `lease_id` 传给 `put()`
- `keepAliveOnce()` 是租约续约，不会自动后台循环
- `config.keepalive` 与租约续约没有直接关系

### 7.3 Pipeline

```cpp
using PipelineOp = galay::etcd::EtcdClient::PipelineOp;

std::vector<PipelineOp> ops;
ops.push_back(PipelineOp::Get("/batch/key1"));
ops.push_back(PipelineOp::Put("/batch/key2", "value2"));
ops.push_back(PipelineOp::Del("/batch/key3"));

if (auto result = client.pipeline(std::move(ops)); !result) {
    std::cerr << result.error().message() << '\n';
    return;
}

for (const auto& item : result.value()) {
    // item.type / item.kvs / item.deleted_count
}
```

当前 pipeline 的精确定义：

- 底层是单次 `POST /v3/kv/txn`
- 没有公开 compare 条件
- 没有公开 failure 分支
- 成功时逐项返回解析结果；错误时以整个调用失败的形式返回

如果你需要 compare-and-swap、分布式锁或条件事务，请不要把当前 pipeline 文档误读为“已经支持”。

## 8. 错误诊断

建议在失败时至少记录：

- `result.error().type()`
- `result.error().message()`

## 9. 生产使用建议

- 即便公开示例 / 测试 / benchmark 在省略 `argv[1]` 时会回落到 `http://127.0.0.1:2379`，生产环境仍应显式传 endpoint
- 多端点集群测试与集群 wrapper 集成测试使用 `GALAY_ETCD_ENDPOINTS=http://127.0.0.1:2379,http://127.0.0.1:22379,...`
- 单 client 保持串行使用；共享场景通过 pool 为每次操作取得独占租约
- 为大前缀查询设置 `limit`
- 为同步与异步请求都设置显式超时
- `EtcdClusterClient` 与 `AsyncEtcdClusterClient` 提供 `withClient()` 自动连接/归还便利层；透明 retry、failover 仍由业务层决定
- 需要 TLS / 认证 / 条件事务时，先确认当前公开 API 并未实现这些能力
- 如果要接事件订阅，优先使用 `AsyncEtcdClient::watch(...)`
