## 4. 测试场景状态

### 4.1 TLS 握手性能

计划指标：

- 握手次数/秒
- 握手延迟

本次状态：blocked。

原因：

- 当前构建不是 Release。
- `GALAY_BUILD_BENCHMARKS=OFF`，未生成 `benchmark_ssl_tls_server_throughput` / `benchmark_ssl_tls_client_throughput`。
- 8443 端口没有已运行的 TLS benchmark 服务端，不能安全运行 `openssl s_time` 性能测试。

补跑命令：

```bash
rtk cmake -S .. -B ../build-release -DCMAKE_BUILD_TYPE=Release -DGALAY_BUILD_BENCHMARKS=ON -DGALAY_BUILD_SSL=ON
rtk cmake --build ../build-release --parallel --target benchmark_ssl_tls_server_throughput benchmark_ssl_tls_client_throughput benchmark_ssl_tls_steady_state
rtk ../build-release/benchmark/cpp/ssl/benchmark_ssl_tls_server_throughput 8443 ../test/cpp/ssl/certs/server.crt ../test/cpp/ssl/certs/server.key 4096 4
rtk openssl s_time -connect 127.0.0.1:8443 -time 30 -www /
```

### 4.2 TLS 吞吐

计划指标：

- Requests/sec
- Throughput MB/s
- Total errors
- 可选 SSL IO stats

本次状态：blocked。

原因：

- Galay SSL benchmark 客户端未构建。
- 没有运行中的 Galay SSL benchmark server。
- `wrk` 可用，但没有 HTTPS 静态文件/echo 服务端，不能产生有效吞吐数据。

补跑命令：

```bash
rtk ../build-release/benchmark/cpp/ssl/benchmark_ssl_tls_server_throughput 8443 ../test/cpp/ssl/certs/server.crt ../test/cpp/ssl/certs/server.key 4096 4
rtk ../build-release/benchmark/cpp/ssl/benchmark_ssl_tls_client_throughput 127.0.0.1 8443 50 200 47 4
rtk env GALAY_SSL_STATS=1 ../build-release/benchmark/cpp/ssl/benchmark_ssl_tls_client_throughput 127.0.0.1 8443 50 200 47 4
```

如果要按计划跑 HTTPS HTTP 吞吐，需要先准备能响应 `/files/1kb.bin` 的 HTTPS 服务端：

```bash
rtk wrk -t4 -c50 -d30s --latency https://127.0.0.1:8443/files/1kb.bin
```

### 4.3 Cipher 对比

计划对比：

- TLS 1.3 `TLS_AES_128_GCM_SHA256`
- TLS 1.2 `ECDHE-RSA-AES128-GCM-SHA256`

本次状态：只完成 OpenSSL cipher 支持探测，性能对比 blocked。

OpenSSL 3.6.2 支持以下待测 cipher：

| TLS 版本 | Cipher | 探测状态 |
|----------|--------|----------|
| TLS 1.3 | `TLS_AES_128_GCM_SHA256` | supported |
| TLS 1.2 | `ECDHE-RSA-AES128-GCM-SHA256` | supported |

补跑命令：

```bash
rtk openssl ciphers -s -v -tls1_3 -ciphersuites TLS_AES_128_GCM_SHA256
rtk openssl ciphers -s -v -tls1_2 'ECDHE-RSA-AES128-GCM-SHA256'
```

需要 Release benchmark server/client 后，才能补充同负载下的 TLS 1.2 vs TLS 1.3 真实性能对比。

## 2026-07-04 同语言 TLS 握手竞品对比

**原始数据**: [ssl_same_language_compare_2026-07-04.csv](./benchmark_data/ssl_same_language_compare_2026-07-04.csv)  
**原始输出**: [openssl_stime_new_vs_galay_tls_2026-07-04.txt](./benchmark_data/openssl_stime_new_vs_galay_tls_2026-07-04.txt)，[openssl_stime_new_vs_openssl_s_server_2026-07-04.txt](./benchmark_data/openssl_stime_new_vs_openssl_s_server_2026-07-04.txt)

执行口径：

```bash
rtk ../build-release/benchmark/cpp/ssl/benchmark_ssl_tls_server_throughput 18443 ../test/cpp/ssl/certs/server.crt ../test/cpp/ssl/certs/server.key 4096 4
rtk /opt/homebrew/bin/openssl s_time -connect 127.0.0.1:18443 -new -tls1_3 -time 5 -no-CAfile -no-CApath -no-CAstore

rtk /opt/homebrew/bin/openssl s_server -quiet -tls1_3 -accept 127.0.0.1:18444 -cert ../test/cpp/ssl/certs/server.crt -key ../test/cpp/ssl/certs/server.key
rtk /opt/homebrew/bin/openssl s_time -connect 127.0.0.1:18444 -new -tls1_3 -time 5 -no-CAfile -no-CApath -no-CAstore
```

| implementation | connections | real seconds | conn/s(real) | conn/user-s | 状态 |
|---|---:|---:|---:|---:|---|
| galay TLS server | 6,839 | 6 | 1,139.83 | 5,065.93 | ok |
| OpenSSL `s_server` | 7,638 | 6 | 1,273.00 | 4,992.16 | ok |

按 real seconds 统计，Galay TLS server 是 OpenSSL `s_server` 的 `89.54%`；按 `s_time` user-second rate 统计，Galay 为 `101.48%`。本节只覆盖 TLS 1.3 新建连接握手，不覆盖 TLS echo 吞吐或 HTTPS 静态文件。

`nginx` 与 Boost.Asio SSL 均已尝试安装依赖；Homebrew bottle 下载被 reset，当前环境无法补跑对应竞品数据。
