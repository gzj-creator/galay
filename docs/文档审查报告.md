## 二、本次新发现的问题

### 🔴 P0-6：模块文档硬编码开发者私有路径（3 个模块）

多个模块的"快速开始"和性能测试文档中，CMake 命令直接写死了开发者本机的 `/Users/gongzhijie/...` 路径，读者复制即失败。

| 文件 | 行号 | 硬编码内容 |
|------|------|-----------|
| `cpp/modules/redis/00-快速开始.md` | 28 | `-DCMAKE_PREFIX_PATH=/Users/gongzhijie/Desktop/projects/git/.galay-prefix/latest` |
| `cpp/modules/ssl/00-快速开始.md` | 25 | `-DCMAKE_PREFIX_PATH=/Users/gongzhijie/Desktop/projects/git/galay-kernel/_install-smoke-344` |
| `cpp/modules/http/00-快速开始.md` | 19, 34 | 同 redis，出现 2 次 |
| `cpp/modules/http/05-性能测试.md` | 195, 225 | `.galay-prefix/latest`、`build-ssl` |
| `cpp/modules/http/07-常见问题.md` | 11 | `.galay-prefix/latest` |

**影响**：新用户照抄命令必然构建失败，且暴露了开发者的本地目录结构。

**建议**：统一替换为占位符 `<galay-install-prefix>` 或文档约定的安装路径，并在首次出现处说明含义。

---

### 🔴 P0-7：异步示例普遍缺少 main 函数与调度入口（4 个模块）

多个"快速开始"中的协程代码示例只给了 `Task<void> run(...)`，却没有展示如何创建 `IOScheduler`、如何 `co_await` 或 `spawn` 启动协程、`main` 函数长什么样。新用户无法据此写出可编译运行的程序。

| 文件 | 问题 |
|------|------|
| `cpp/modules/redis/00-快速开始.md` | `run(IOScheduler*)` 协程后无 main，未展示 scheduler 创建 |
| `cpp/modules/mysql/00-快速开始.md` | 异步示例无 main、无调度入口 |
| `cpp/modules/etcd/00-快速开始.md` | 异步示例无 main、无 `Task` 的 include |
| `cpp/modules/http/00-快速开始.md` | **全文无任何 C++ 代码示例**，只有命令行 |

**建议**：每个"快速开始"必须包含一个完整的、可复制粘贴即跑的示例（含 include、main、scheduler 创建、协程启动）。

### 🟡 P1-7：plans 目录存在 5 组语义重复主题

同一主题被拆成多个计划文件，边界不清，读者不知该看哪个。日期前缀格式本身合规，但内容治理混乱。

| 主题 | 冲突文件 | 问题 |
|------|---------|------|
| RPC 生产就绪 | `06-21-rpc-production-complete-plan` + `...-readiness-phase1` + `...-readiness` | 同日 3 文件，complete/readiness/phase1 边界模糊 |
| 示例测试风格 | `06-14-examples-tests-style-convergence` + `...-style-guide` | 同日同主题，convergence 与 guide 语义重叠 |
| HTTP/2 静态快路径 | `06-20-http2-static-fast-path-plan` + `...-http2-static-file-fast-path` | static fast path / static file fast path 高度近似 |
| kernel C accept | `06-22-kernel-c-tcp-accept-demo` + `06-23-kernel-c-accept-callback` | 紧邻主题分两天记录，应标注演进关系 |
| C 协程运行时 | `06-22-c-api-ffi-coroutine-plan` + `06-27-c-coro-runtime` + `06-28-direct-c-coroutine-async-c-abi` | 同主题跨日 3 文件，无阶段标记 |

此外，后缀约定不统一：同目录混用 `-plan`、`-design`、`-audit`、`-remediation`、`-guide`、`-convergence`、`-hardening`、`-cleanup` 等 8 种后缀，无统一语义约定。

**建议**：建立 plans 文档的命名约定（如统一用 `-design` 表设计、`-plan` 表实施计划），对重复主题做归并或加"前置/后续"链接。
