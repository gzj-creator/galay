## 2. `examples/` 中的真实示例

统一构建命令：

```bash
cmake -S . -B build-examples \
  -DBUILD_MODULE_TESTS=OFF \
  -DBUILD_EXAMPLES=ON \
  -DBUILD_TESTING=OFF
```

### 2.1 `E1-BasicUsage`

- 源文件：`examples/include/e1_basic.cpp`
- 覆盖模块：`StringUtils`、`System`
- 适合场景：验证 umbrella header 接入是否打通
- 额外前置：无额外第三方头依赖

```bash
cmake --build build-examples --target E1-BasicUsage --parallel
./build-examples/examples/E1-BasicUsage
```

### 2.2 `E1-BasicUsageImport`

- 源文件：`examples/import/e1_basic.cpp`
- 覆盖模块：`StringUtils`、`System`
- 适合场景：验证 `import galay.utils;` 的最小消费路径
- 额外前置：CMake `>= 3.28`、支持模块扫描的生成器/编译器，并显式开启 `BUILD_MODULE_TESTS`

```bash
cmake --build build-examples --target E1-BasicUsageImport --parallel
./build-examples/examples/E1-BasicUsageImport
```

## 4. 模块导入示例

- 源文件：`test/import_smoke.cpp`
- target：`test_import_smoke`
- 环境变量：无

前置条件：

- CMake `>= 3.28`
- `Ninja` 或 `Visual Studio`
- 非 `AppleClang`
- 无额外第三方头依赖

当前烟雾测试覆盖：

- `import galay.utils;`
- core、process、tool、algorithm、app/config、encoding/crypto 的基础消费路径
- 限流器仍通过 `galay-utils/tool/rate_limiter.hpp` 显式包含

## 5. 本页不再做的事情

- 不把没有对应源文件的长片段标成“官方示例”
- `.ini` 使用 `IniParser`，`.conf` 使用 `ConfigParser`，`.toml` 使用 `TomlParser`
- 不用不存在的 target 或目录名描述运行方式
